## 1. 화면 구성

```
┌───────────────────────────────────────────┐
│ 🗂 내 보드                          (나)   │
├───────────────────────────────────────────┤
│ [🔍 보드 검색            ]  [+ 새 보드]     │
├───────────────────────────────────────────┤
│ (전체) 내가 만든 보드  참여 중인 보드         │
├───────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│ │ owner    │ │ guest    │ │  + 새 보드 │    │
│ │          │ │          │ │  만들기   │    │
│ │여행 보드  │ │스터디 보드│ │          │    │
│ │참여3·2h전 │ │참여5·어제 │ │          │    │
│ └──────────┘ └──────────┘ └──────────┘    │
└───────────────────────────────────────────┘
```

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 서비스 로고/이름, 본인 계정 아바타(우측) |
| 검색 + 새 보드 버튼 | 보드 이름 검색 입력창, "새 보드" 버튼 |
| 필터 탭 | 전체 / 내가 만든 보드 / 참여 중인 보드 |
| 보드 카드 그리드 | 보드별 카드: 역할 뱃지, 보드 이름, 참여자 수, 마지막 활동 시각 |
| 새 보드 카드 | 그리드 마지막 칸에 점선 카드로 배치, 클릭 시 보드 생성 모달 오픈 |

---

## 2. 확정 사항

### 2-1. 목록 노출 기준
**guest 이상 권한을 받은 보드만 목록에 노출한다.** 초대 링크로 잠깐 들어와 watcher로만 머문 보드는 "내 보드 목록"에 남기지 않는다. 목록이 지저분해지는 것을 방지하기 위함.

```
목록 노출 조건: board_member.role IN ('owner', 'guest')
watcher로만 참여한 보드는 미노출
```

### 2-2. 보드 썸네일
실제 캔버스 내용을 렌더링한 미리보기 대신, **보드의 `background_color` 값을 그대로 카드 상단 색상 블록으로 사용**한다. 실시간 캔버스 캡처 로직은 구현 비용 대비 효용이 낮아 MVP 범위에서 제외.

### 2-3. 정렬 기준
**최근 활동순(최근 수정된 보드가 상단)**으로 정렬한다. 기준 필드는 `BOARD.updated_at`.

---

## 3. 카드 표시 정보

| 항목 | 데이터 출처 |
|---|---|
| 역할 뱃지 (owner/guest) | `board_member.role` |
| 보드 이름 | `BOARD.title` |
| 참여자 수 | `board_member` 중 해당 board_id 카운트 |
| 마지막 활동 시각 | `BOARD.updated_at` (상대 시간 표기: "2시간 전", "어제" 등) |
| 카드 배경색 | `BOARD.background_color` |
