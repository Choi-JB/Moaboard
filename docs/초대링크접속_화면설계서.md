## 1. 정상 접속 화면 (완충 화면)

링크 클릭 즉시 보드로 자동 입장시키지 않고, 어떤 보드인지 확인 후 입장하도록 한 단계 거친다.

```
┌───────────────────────────┐
│                             │
│     [보드 미리보기]          │
│                             │
│   여행 브레인스토밍 보드      │
│   민지님이 초대하였습니다     │
│      ● ● ●  3명 참여 중      │
│                             │
│    [ 보드 입장하기 ]         │
│                             │
│  읽기 전용(watcher)으로      │
│  입장해요. 편집하려면 보드     │
│  안에서 권한을 요청하세요     │
│                             │
└───────────────────────────┘
```

| 요소 | 설명 |
|---|---|
| 보드 미리보기 | `background_color` 기반 색상 블록 (보드 목록 화면과 톤 통일) |
| 보드 이름 | `BOARD.title` |
| 초대자 문구 | "{owner 닉네임}님이 초대하였습니다" — `BOARD.owner_id` → 닉네임 조회 |
| 참여자 정보 | 아바타 미리보기 + 인원 수 |
| 입장 버튼 | 클릭 시에만 입장 처리 (자동 입장 아님) |
| 안내 문구 | watcher로 입장됨을 명시. 편집 권한 요청 문구는 노출하되, **요청 기능 자체는 이번 범위에서 구현하지 않음** (향후 보드 메인 화면의 참여자 UI에서 추가 예정) |

### 처리 흐름
```
"보드 입장하기" 클릭
→ 세션 확인 (로그인_계정연동_화면설계서.md 3-1 로직과 동일)
   - 세션 없음 → Anonymous Auth 신규 세션 생성
   - 세션 있음 → 기존 user_id 사용
→ board_member에 watcher로 upsert
→ /board/:id 로 이동
```

## 2. 에러 화면 (링크 무효 / 보드 삭제)

```
┌───────────────────────────┐
│                             │
│         ⛔ (아이콘)          │
│                             │
│  유효하지 않은 초대 링크예요   │
│  링크가 잘못되었거나           │
│  보드가 삭제되었을 수 있어요    │
│                             │
│   [내 보드 목록으로 이동]      │
│                             │
└───────────────────────────┘
```

| 항목 | 내용 |
|---|---|
| 노출 조건 | `invite_token`으로 조회된 보드가 없거나, `BOARD.deleted_at`이 채워진 경우 |
| 메시지 | 링크 무효와 보드 삭제를 구분하지 않고 하나의 메시지로 통합 (사용자 입장에서 결과가 동일하므로 구분 실익 낮음) |
| 하단 버튼 | 로그인 상태면 "내 보드 목록으로 이동", 비로그인 상태면 랜딩 화면(`/`)으로 이동 |
