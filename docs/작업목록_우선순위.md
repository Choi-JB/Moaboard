# 남은 작업 목록 — 우선순위

2026-07-21 기준, `README.md`의 "남은 과제"에 사용자가 추가로 원하는 3개 항목을 더해 총 10개.
빠르게 끝날 것으로 예상되는 순서대로 정렬. (실제 착수 전 재평가될 수 있음)

| 순위 | 항목 | 비고 |
|---|---|---|
| 1 | 객체 리스트에서 객체 선택 시 강조 표시 | 로컬 상태 하나(`selectedObjectId`) 추가 + 조건부 스타일. 새 API/DB 불필요 |
| 2 | 참여자 아이콘 클릭 시 정보 카드 표시 | `ParticipantAvatarStack`에 이미 nickname/role 데이터 있음. 클릭 시 여는 팝오버 컴포넌트만 추가 |
| 3 | DB에서 삭제된 보드/오브젝트 자동 처리 | `supabase/schema.sql`에 board_objects용 pg_cron 코드가 이미 주석으로 준비됨. 주석 해제 + BOARD 테이블용 추가 |
| 4 | 익명 유저 자동 처리 (pg_cron 잡) | 3번과 유사한 SQL 작업. 비활성 판단 기준(기간)을 새로 정해야 함 |
| 5 | props들 store로 이관 | 현재 prop 체인은 `BoardPage → Canvas → MemoObject/ImageObject` 2단계. 기계적 리팩터링 |
| 6 | 오브젝트 사이즈 조절 | 기존 드래그/브로드캐스트/자동저장 패턴 재사용 가능. 리사이즈 핸들 UI만 추가 |
| 7 | 캔버스 배율 조정 (줌 버튼 + 현재 배율 표시) | 줌 UI는 쉬우나, 배율 변경 시 클릭 생성 위치·드래그 좌표를 전부 보정해야 함 |
| 8 | 익명 로그인 유저 구글 로그인 연동 (AccountLinkPopover) | Supabase identity linking API 필요, 인증 엣지 케이스(이메일 충돌 등) 고려 |
| 9 | UI 꾸미기 (전체적으로) | 범위가 "전체"라 명확한 끝이 없음. 착수 전 완료 기준을 먼저 좁혀야 함 |
| 10 | 신규 오브젝트 추가 (영상, 달력, 카드, 주사위 등) | 타입마다 데이터 모델 + 렌더링 + 개별 동작을 새로 설계해야 하는 여러 개의 미니 기능. 가장 큼 |

## 진행 방식
- 이 프로젝트는 2주차부터 사용자가 직접 코드를 작성하고, Claude는 가이드/설명/리뷰만 담당하는 방식으로 진행 중.
- 항목 착수 시 이 표의 순서를 그대로 따르지 않아도 됨 — 상황에 따라 재조정 가능.
