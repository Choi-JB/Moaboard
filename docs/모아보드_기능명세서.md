## 0. 서비스 개요

여러 명이 실시간으로 동시 접속하여 메모, 이미지 등을 자유롭게 배치하고 꾸밀 수 있는 온라인 화이트보드 서비스. 링크 초대 기반으로 참여하며, 역할(owner/guest/watcher)에 따라 편집 권한이 달라진다.

## FT-01. 멀티 접속 (초대 링크)

### 기능 설명
보드 소유자가 초대 링크를 생성하고, 이 링크로 접속한 사용자는 기본적으로 watcher(읽기 전용) 권한으로 보드에 입장한다.

### 사용자 시나리오
```
1. owner가 보드 상단 메뉴에서 "초대 링크 생성" 클릭
2. 시스템이 invite_token을 발급, 공유 가능한 URL 생성
3. 손님이 링크로 접속
4. 시스템이 해당 유저를 board_member에 role='watcher'로 등록
5. 손님은 읽기 전용 상태로 보드 진입
```

### 처리 로직
- 링크 접속 시 로그인 여부와 무관하게 우선 입장 가능 (Supabase Anonymous Auth로 익명 세션 발급)
- 기존에 이미 board_member로 등록된 유저(재접속)라면 기존 role 유지, 신규 유저라면 watcher로 신규 등록

### 예외 처리
| 상황 | 동작 |
|---|---|
| 유효하지 않은 invite_token | "존재하지 않는 보드입니다" 안내 후 이동 차단 |
| 보드가 삭제된 상태 | 동일 안내 |

## FT-02. 권한 설정

### 기능 설명
owner가 접속자 중 한 명을 선택해 개별로 권한(guest/watcher)을 부여하거나 회수할 수 있다. 화면 상단에 위치.

### 권한 정의

| 역할 | 객체 생성 | 잠기지 않은 객체 수정/삭제 | 잠긴 객체 접근 | 잠금 설정/해제 | 보드 설정 변경 | 권한 부여 |
|---|---|---|---|---|---|---|
| owner | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| guest | ✅ | ✅ (본인/타인 구분 없음) | ❌ | ❌ | ❌ | ❌ |
| watcher | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |

### 사용자 시나리오
```
1. owner가 상단 참여자 목록에서 특정 유저 선택
2. role 변경 옵션(guest ↔ watcher) 노출
3. 변경 즉시 board_member.role 업데이트
4. 대상 유저 화면에도 실시간 반영 (권한 변경 알림 또는 UI 자동 갱신)
```

### 비로그인(익명) 유저 처리
- Supabase Anonymous Auth로 익명 로그인 → `is_anonymous = true`로 구분
- 비로그인 상태에서도 guest 권한 부여 및 생성/수정/삭제 가능
- **역할은 익명 세션(브라우저) 기준으로 유지**되며, 탭을 닫았다 다시 들어와도 동일 세션이면 role 유지
- 권한 회수는 owner가 명시적으로 watcher로 변경할 때만 발생 (자동 만료 없음)
- 유일한 예외: 브라우저 캐시/localStorage 삭제 시 익명 세션 자체가 사라져 재접속 시 watcher로 신규 등록됨 (자연스러운 동작으로 처리)

## FT-03. 객체 (생성 / 이동 / 삭제 / 잠금)

### 기능 설명
보드 위에 메모지, 이미지 등의 객체를 생성하고 자유롭게 배치·삭제·잠금할 수 있다. 화면 하단에 생성 메뉴 위치.

### 상세 정의

| 동작 | 설명 |
|---|---|
| 종류 | 메모지(memo), 이미지(image) |
| 생성 | 하단 메뉴에서 종류 선택 → 보드 클릭 시 해당 위치에 생성 |
| 이동 | 드래그로 보드 내 자유 배치 |
| 삭제 | 즉시 삭제 아님 → `deleted_at` 기록 (휴지통 처리), 다음날 00시 DB에서 완전 삭제 |
| 잠금 | 자물쇠 아이콘 표시, `is_locked = true` 시 owner 외 접근/수정 불가 |

### 처리 로직
- 잠금 설정/해제는 owner만 가능
- 삭제된(`deleted_at` not null) 객체는 화면에서 즉시 숨김 처리, 실제 DB 삭제는 익일 00시 배치(pg_cron)로 처리
- 객체 생성 시 보드 내 현재 활성 객체 수(`deleted_at is null`)가 200개 이상이면 생성 차단

### 예외 처리
| 상황 | 동작 |
|---|---|
| 잠긴 객체를 guest가 드래그 시도 | 이동 불가, "owner만 수정 가능" 안내 |
| 보드당 200개 초과 시 생성 시도 | 생성 버튼 비활성화 또는 안내 메시지 |
| watcher가 생성/이동/삭제 시도 | 관련 UI 자체 미노출 또는 클릭 무시 |

## FT-04. 현재 객체 리스트 표시

### 기능 설명
현재 보드에 있는 모든 객체를 리스트 형태로 표시. 우상단 버튼 클릭 시 노출.

### 상세 정의
- 리스트에는 활성 객체(`deleted_at is null`)만 표시
- 필터: 모두 / 내것 / 타인것
- 필터 적용 시 해당하지 않는 객체는 보드 화면에서 흐릿하게 처리(반투명)

### 사용자 시나리오
```
1. 우상단 버튼 클릭 → 객체 리스트 패널 오픈
2. 필터 선택(모두/내것/타인것)
3. 필터 조건에 해당하지 않는 객체는 보드에서 흐림 처리
```

## FT-05. 자동 저장

### 기능 설명
사용자의 조작이 "종료"되는 시점에 자동으로 DB에 저장한다. 실시간 브로드캐스트(화면 반영)와 DB 저장(영속화)은 분리하여 처리한다.

### 저장 시점 정의

| 동작 | 저장 시점 |
|---|---|
| 메모 생성/편집 | 타이핑 종료 후 500ms 무입력 시 (디바운스) |
| 객체 이동 | 드래그 종료(mouseup) 시점 |
| 객체 생성/삭제 | 즉시 |
| 보드 설정 변경 | 즉시 |

### 처리 로직
- 드래그/타이핑 중에는 로컬 상태 갱신 + Realtime 브로드캐스트만 수행
- 위 표의 "종료 시점"에만 실제 DB Write 발생

## 2. 제약사항

| 항목 | 내용 |
|---|---|
| 객체 생성 제한 | 보드당 활성 객체 200개 이하 |
| 삭제 유예기간 | 삭제 결정 후 다음날 00시 DB에서 실제 삭제 (pg_cron) |
| 비로그인 유저 | Anonymous Auth 사용, `is_anonymous`로 구분. 익명 세션 기준으로 role 유지, owner가 수동 회수하기 전까지 상실되지 않음 |
