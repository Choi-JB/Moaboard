## 1. 화면 구성 (모달)

```
┌───────────────────────────────┐
│ 새 보드 만들기              ✕  │
├───────────────────────────────┤
│ 보드 이름                       │
│ [예: 여행 브레인스토밍        ] │
│                                 │
│ 캔버스 크기                     │
│ [FHD]      [QHD]      [UHD]    │
│ 1920×1080  2560×1440  3840×2160│
│                                 │
│ 배경색                          │
│ ● ● ● ●                        │
│                                 │
│  [취소]         [만들기]        │
└───────────────────────────────┘
```

| 요소 | 설명 |
|---|---|
| 보드 이름 | 텍스트 입력, 플레이스홀더로 예시 표시 |
| 캔버스 크기 | FHD/QHD/UHD 선택, 실제 해상도 수치 함께 표기 |
| 배경색 | 색상 원형 스와치 중 선택 |
| 취소/만들기 | 하단 버튼 |

## 2. 확정 사항

### 2-1. 보드 이름 미입력 시 처리
보드 이름을 비워둔 채 "만들기"를 눌러도 생성이 차단되지 않는다. `BOARD.title` 기본값("제목 없는 보드")으로 생성되며, 이름은 이후 보드 메인 화면의 설정 패널에서 언제든 수정할 수 있다.

### 2-2. 생성 후 이동
"만들기" 클릭 시 보드 목록에 머무르지 않고, **생성된 보드로 즉시 이동**한다.
```
"만들기" 클릭 → BOARD 레코드 생성 → board_member에 본인을 owner로 등록 → /board/:id 로 이동
```

### 2-3. 초대 링크 자동 발급
보드 생성 시점에 `invite_token`도 함께 자동 발급한다. 별도로 "초대 링크 생성" 버튼을 누르는 절차 없이, 생성 직후 설정 패널에서 바로 링크를 복사할 수 있다.

---

## 3. 데이터 처리 순서

```
1. 사용자 입력: title(선택), canvas_size, background_color
2. BOARD 테이블에 insert
   - title: 입력값 또는 기본값 '제목 없는 보드'
   - canvas_size, background_color: 선택값
   - owner_id: 현재 로그인 유저
   - invite_token: 자동 생성(uuid 또는 랜덤 문자열)
3. board_member 테이블에 insert
   - board_id: 방금 생성된 보드 id
   - user_id: 현재 로그인 유저
   - role: 'owner'
4. /board/:id 로 라우팅
```
