## 1. 전체 화면 목록 (IA)

| 화면 | 경로(예상) | 설명 |

## 2. 보드 메인 화면 레이아웃

```
┌─────────────────────────────────────────────────┐
│ [설정]        (●오너)(●닉네임)(●닉네임)           │  ← 상단바 48px
├─────────────────────────────────────────┬───────┤
│                                           │       │
│                                           │ 객체  │
│              캔버스 영역                  │ 리스트│
│         (메모/이미지 자유 배치)            │ 서랍  │
│                                           │(230px)│
│                                           │       │
│              ⬤ [메모] [이미지] ⬤           │       │
│           (하단 테두리에서 띄운 pill 메뉴)   │       │
└─────────────────────────────────────────┴───────┘
                                              ▲
                                     화살표 탭(열기/닫기)
```

### 2-1. 상단바 (48px, 고정)

| 위치 | 요소 | 설명 |
|---|---|---|
| 좌측 | 설정 버튼 | 클릭 시 좌상단 근처에 설정 패널 오픈 (보드 크기 등) |
| 중앙 | 참여자 아바타 스택 | 구글 문서 방식의 겹친 원형 아바타. 역할별 색상 구분(owner/guest/watcher). **표시 대상은 `board_member` 전체가 아니라 Presence 기반 "현재 접속 중인 사람"만** (상세: 실시간동기화_상세설계서.md 2-4) |
| 우측 | (여백) | 좌우 대칭을 위한 빈 공간 |

**참여자 아바타 클릭 시 (owner만 동작)**
- 드롭다운 노출: 현재 권한 표시(체크) + guest ↔ watcher 전환 + 내보내기(강퇴)
- **본인(owner) 아바타는 클릭 비활성** — 소유권 이전, 본인 권한 낮추기 기능은 이번 범위에서 제외
- guest/watcher 본인은 자기 아바타를 클릭해도 드롭다운 없음 (owner만 권한 변경 가능)

### 2-2. 캔버스 영역

- 상단바(48px)와 하단 메뉴 사이 전체 공간
- 메모(memo), 이미지(image) 객체를 드래그로 자유 배치
- 잠긴 객체는 자물쇠 아이콘 표시, owner 외에는 접근 불가
- 객체 리스트 서랍이 열리면 우측 230px만큼 캔버스 폭이 줄어듦 (겹치지 않음)

### 2-3. 하단 객체 생성 메뉴

- 화면 하단 테두리에서 띄운 pill(캡슐) 형태, 중앙 정렬
- 현재 항목: 메모, 이미지
- 향후 객체 타입(지도, 달력, 카드 등) 추가 시 "더보기(+)" 버튼으로 확장 예정

### 2-4. 객체 리스트 서랍 (우측)

- 기본 상태: 닫힘, 화면 우측 끝에 화살표 탭만 노출(`<`)
- 클릭 시 우측에서 230px 폭으로 슬라이드 인/아웃
- 열림 상태에서 탭 아이콘은 반대 방향(`>`)으로 전환되어 닫기 동작 표시

**필터 탭**
| 필터 | 설명 |
|---|---|
| 전체 | 보드 내 모든 활성 객체 표시 |
| 내것 | 본인(`created_by`)이 만든 객체만 |
| (참여자별 닉네임) | 전체/내것 외 나머지 참여자는 각자의 닉네임으로 개별 필터 탭 생성 (예: "민지", "수현") |

- 필터 선택 시 캔버스에서 조건에 해당하지 않는 객체는 반투명 처리
- 리스트 항목: 타입 아이콘 + 객체 이름/내용 요약 + 생성자 닉네임 + 생성 시점, 잠긴 객체는 자물쇠 아이콘 표시
- 리스트 항목 클릭 시 동작은 MVP 범위 외 (향후 계획: 캔버스 포커스 이동 + 줌 강조)

## 3. 권한 변경 드롭다운 (상세)

```
┌─────────────────────────┐
│ 민지님 권한               │
├─────────────────────────┤
│ ✓ guest · 편집 가능       │
│   watcher · 읽기 전용     │
├─────────────────────────┤
│ 🚫 내보내기               │
└─────────────────────────┘
```

| 항목 | 내용 |
|---|---|
| 진입 방식 | owner가 참여자 아바타 클릭 |
| 표시 정보 | 대상자 닉네임, 현재 권한(체크 표시) |
| 변경 가능 옵션 | guest ↔ watcher |
| 위험 동작 | 내보내기(강퇴) — 구분선 아래 별도 배치, 위험색 표시 |
| 제외 범위 | 소유권 이전, 본인 권한 스스로 낮추기 — 이번 범위에서 다루지 않음 |

## 3-1. 설정 패널 (상세)

```
┌───────────────────────────┐
│ 보드 설정                   │
├───────────────────────────┤
│ 보드 이름                   │
│ [여행 브레인스토밍 보드    ] │
├───────────────────────────┤
│ 캔버스 크기                 │
│ [FHD] [QHD] [UHD]          │
├───────────────────────────┤
│ 배경                       │
│ ● ● ● ●                    │
├───────────────────────────┤
│ 초대 링크                   │
│ [whiteboard.app/invite/... ] [복사] │
├───────────────────────────┤
│ 🗑 보드 삭제                │
└───────────────────────────┘
```

| 항목 | 내용 |
|---|---|
| 진입 방식 | 상단바 좌측 "설정" 버튼 클릭 → 좌상단 근처에 팝업 패널 노출 |
| **노출 권한** | **owner만 설정 버튼이 보임. guest/watcher에게는 버튼 자체가 노출되지 않음** (초대 링크 등 owner 전용 정보 보호 목적) |
| 구성 항목 | 보드 이름 편집, 캔버스 크기(FHD/QHD/UHD), 배경색 선택, 초대 링크 복사, 보드 삭제 |
| 제외 항목 | 테마 선택 — 배경색과 역할이 중복되어 이번 범위에서 제외 |

**보드 삭제 플로우**
```
"보드 삭제" 버튼 클릭
→ 확인 모달 노출: "정말 삭제하시겠습니까? 삭제된 보드는 복구할 수 없습니다."
→ [취소] [삭제] 버튼
→ [삭제] 선택 시에만 실제 삭제 처리
```
- 실수 방지를 위해 버튼 클릭 즉시 삭제되지 않고 반드시 확인 모달을 거침

## 4. 이번 범위에서 제외한 것 (확인 완료)
- 보드 소유권 이전 기능
- guest/owner가 스스로 자기 권한을 낮추는 기능
- 객체 리스트 클릭 시 캔버스 포커스 이동/줌 (향후 계획으로 이관)
- 테마 선택 기능 (배경색 선택과 역할 중복으로 제외)
